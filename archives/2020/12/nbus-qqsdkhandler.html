NBus 之 QQSDKHandler

发表于 2020-12-06 更新于 2021-01-16 分类于 program 阅读次数：

QQ SDK 的接入问题不少，本 Handler 基于 QQ SDK 3.3.9 实现。

General

调用 TencentOAuth.init(appId:enableUniveralLink:universalLink:delegate:) 注册 QQ SDK，调用 QQApiInterface.startLog(_:) 记录日志。Helper 处理 QQ 的回调，相关内容下文会进行说明。

LogHandlerProxyType

声明 QQSDKHandler 遵循 LogHandlerProxyType 协议。

ShareHandlerType

在分享流程中：

  1. 调用 isInstalled 判断是否安装 QQ，没有则提前退出。
  2. 调用 canShare(message:to:) 判断 Endpoint 是否支持此类型 Message。例如 Endpoints.QQ.timeline 空间不支持 Messages.file 文件和 Messages.miniProgram 小程序。
  3. 判断 message 的具体类型，创建 SendMessageToQQReq 请求。除了 QQApiMiniProgramObject 小程序类型，其他消息类型和 SendMessageToQQReq 都需要使用带参数标签的指定构造器，否则无效。
  4. 当 Endpoint 是 Endpoints.QQ.friend 好友时，调用 cflag(_:_:) 设置 kQQAPICtrlFlag 参数，调用 QQApiInterface.send(_:) 拉起 QQ 好友分享。
  5. 当 Endpoint 是 Endpoints.QQ.timeline 空间时，调用 QQApiInterface.sendReq(toQZone:) 拉起 QQ 空间分享。
  6. 调用 code 判断是否为 .EQQAPISENDSUCESS，不是则调用 completionHandler(.failure(Error)) 完成分享流程。

OauthHandlerType

在登录流程中：

  1. 调用 isInstalled 判断是否安装 QQ，没有则提前退出。
  2. 调用 oauthHelper.authorize(_:) 拉起 QQ 登录。

OpenURLHandlerType

调用 QQApiInterface.handleOpen(_:delegate:) 和 TencentOAuth.handleOpen(:_) 处理 URL Scheme 回调。

OpenUserActivityHandlerType

调用 QQApiInterface.handleOpenUniversallink(_:delegate:) 和 TencentOAuth.handleUniversalLink(_:) 处理 NSUserActivity 回调。

Helper

创建 Helper 处理 QQ 的回调：

  1. QQSDKHandler 强持有 Helper，所以 Helper 的 master 声明为 weak 避免循环引用。
  2. 处理分享回调时，onResp(_:) 返回 SendMessageToQQResp，调用 response.result 判断是否为 0，调用 master?.shareCompletionHandler?(.success(())) 完成分享回调。
  3. 处理登录回调时，成功则 tencentDidLogin() 被调用，从 oauthHelper 获取 accessToken 和 openId，调用 master?.oauthCompletionHandler?(.success(parameters)) 完成登录回调。失败则 tencentDidNotLogin(_:) 被调用，调用 master?.oauthCompletionHandler?(.failure(Error)) 完成登录回调。

总结

本文通过封装 QQ SDK 做出 QQSDKHandler，实现 QQ 的登录和分享功能。相较于微信 SDK 的统一，QQ SDK 的分享和登录分散在两个地方，处理 URL Scheme 和 NSUserActivity 时需要同时调用，否则无法完成登录流程。这对使用者来说可能造成一定的困扰。
